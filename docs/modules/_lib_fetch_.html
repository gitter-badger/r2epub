Const check_Web_url

  • check_Web_url(address: URL): URL
    • Defined in lib/fetch.ts:68

    Basic sanity check on a URL that is supposed to be used to retrieve a Web Resource.

    The function returns a (possibly slightly modified) version of the URL if everything is fine, or a null value if the input argument is not a URL (but should be used as a filename).

    There might be errors, however, in the case it is a URL.

    The checks are as follows:

    1. Check whether the protocol is http(s). Other protocols are not accepted (actually rejected by fetch, too);
    2. Run the URL through a valid-url check, which looks at the validity of the URL in terms of characters used, for example;
    3. Check that the port (if specified) is in the allowed range, ie, > 1024;
    throws

    if it pretends to be a URL, but it is not acceptable for some reasons.

    Parameters

    • address: URL

      the URL to be checked.

    Returns URL

    • the URL itself (which might be slightly improved by the valid-url method) or null if this is, in fact, not a URL
